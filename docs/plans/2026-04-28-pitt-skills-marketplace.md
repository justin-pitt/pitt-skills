# pitt-skills Marketplace Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a single GitHub repo (`justin-pitt/pitt-skills`) that serves three roles: (1) Justin's personal Claude Code marketplace synced between home/work laptops via symlinks, (2) a shareable bundle for coworkers using Copilot CLI / VS Code Chat / Desktop, (3) a curated catalog of trusted upstream marketplaces.

**Architecture:** `SKILL.md` files are the only authored artifacts. A PowerShell build script (with a Bash CI wrapper) regenerates `.github/instructions/*.instructions.md`, `.github/prompts/*.prompt.md`, and the JSON manifests. An install script symlinks the live tree into `~/.claude/skills/`, `~/.copilot/skills/`, and `~/.copilot/instructions/`, so `git pull` is the only update verb. Single Claude plugin (`pitt-skills`) wraps all skills.

**Tech Stack:** PowerShell 7+, Bash, GitHub Actions, Pester (PS tests), bats-core (Bash tests), `gh` CLI, golden-file testing for build script output.

**Working dir:** `c:\Code\pitt-skills\` (already initialized; main branch holds the design doc commit).

**Reference:** [docs/plans/2026-04-28-pitt-skills-marketplace-design.md](2026-04-28-pitt-skills-marketplace-design.md).

**Background — the shadowing trap (observed during M1 work-laptop install):**

When a user installs `pitt-skills@pitt-skills` while a standalone copy of the same skill sits at `~/.claude/skills/<name>/`, the standalone takes precedence at skill-resolution time. The plugin version is silently shadowed. The work-laptop M1 install reproduced this for `threatconnect-polarity`. The installer must detect this case and surface it loudly — but **must not silently delete user files**. Default behavior is detect-and-warn; the user opts in to backup-and-remove via a `-RemoveShadowing` switch.

**Step 1: Write settings.snippet.json**

```json

      "source": { "source": "git", "url": "https://github.com/obra/superpowers.git" }
```

**Step 4: Write minimal install.ps1**

```powershell
# scripts/install.ps1
[CmdletBinding()]
param(
    [switch]$DotSource,
    [switch]$WhatIf,
    [switch]$Uninstall,
    [switch]$RemoveShadowing,
    [string[]]$Tools = @('claude')
)

function Get-ClaudeHome {
    if ($env:CLAUDE_HOME) { return $env:CLAUDE_HOME }
    return Join-Path $HOME ".claude"
}

function Merge-ClaudeSettings {
    [CmdletBinding()]
    param(
        [Parameter(Mandatory)] [string] $SnippetPath
    )
    $claudeHome = Get-ClaudeHome
    if (-not (Test-Path $claudeHome)) { New-Item -ItemType Directory -Path $claudeHome -Force | Out-Null }
    $settingsPath = Join-Path $claudeHome 'settings.json'
    $snippet = Get-Content $SnippetPath -Raw | ConvertFrom-Json -AsHashtable

    if (Test-Path $settingsPath) {
        Copy-Item $settingsPath "$settingsPath.bak" -Force
        $existing = Get-Content $settingsPath -Raw | ConvertFrom-Json -AsHashtable
    } else {
        $existing = @{}
    }

    foreach ($key in $snippet.Keys) {
        if ($existing.ContainsKey($key) -and $existing[$key] -is [hashtable] -and $snippet[$key] -is [hashtable]) {
            foreach ($subkey in $snippet[$key].Keys) {
                $existing[$key][$subkey] = $snippet[$key][$subkey]
            }
        } else {
            $existing[$key] = $snippet[$key]
        }
    }

    $existing | ConvertTo-Json -Depth 32 | Set-Content $settingsPath
}

function Get-ShadowingSkills {
    [CmdletBinding()]
```

```powershell
function Parse-Frontmatter {
    param([string]$Content)
    if ($Content -notmatch '(?ms)\A---\r?\n(.*?)\r?\n---\r?\n(.*)\z') {
        throw "No YAML frontmatter found"
    }
    # Lightweight YAML parse — only key:value, nested via 2-space indent
    $yaml = $matches[1]
    $body = $matches[2]
    $result = @{}
    $currentKey = $null
    foreach ($line in $yaml -split "`r?`n") {
        if ($line -match '^([a-zA-Z][\w-]*):\s*(.*)$') {
            if ($matches[2] -eq '') {
                $currentKey = $matches[1]
                $result[$currentKey] = @{}
            } else {
                $result[$matches[1]] = $matches[2].Trim('"').Trim("'")
                $currentKey = $null
            }
        } elseif ($line -match '^\s\s([a-zA-Z][\w-]*):\s*(.*)$' -and $currentKey) {
            $val = $matches[2].Trim('"').Trim("'")
            if ($val -eq 'true') { $val = $true }
            elseif ($val -eq 'false') { $val = $false }
            $result[$currentKey][$matches[1]] = $val
        }
    }
    return @{ Frontmatter = $result; Body = $body }
}

$pluginManifest = @{
    name = 'pitt-skills'
    description = "Justin Pitt's personal collection of Claude Code and Copilot skills"
    version = '0.1.0'
    author = @{ name = 'Justin Pitt'; email = 'justin@pittnet.net' }
}
```

```powershell
function New-DirectorySymlink {
    param([string]$Link, [string]$Target)
    $parent = Split-Path $Link -Parent
    if (-not (Test-Path $parent)) { New-Item -ItemType Directory -Path $parent -Force | Out-Null }
    if (Test-Path $Link) { Remove-Item $Link -Recurse -Force }
```

```bash
#!/usr/bin/env bash
# scripts/install.sh — symlink-based installer for macOS / Linux
set -euo pipefail

REPO_ROOT="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
TOOLS="claude,copilotCli,vscode"
WHATIF=0
UNINSTALL=0

while [[ $# -gt 0 ]]; do
    case "$1" in
        --tools) TOOLS="$2"; shift 2 ;;
        --what-if) WHATIF=1; shift ;;
        --uninstall) UNINSTALL=1; shift ;;
        *) echo "Unknown option: $1" >&2; exit 1 ;;
    esac
done

merge_claude_settings() {
    local claude_home="$HOME/.claude"
    mkdir -p "$claude_home"
    local settings="$claude_home/settings.json"
    local snippet="$REPO_ROOT/settings.snippet.json"
    if [[ -f "$settings" ]]; then
        cp "$settings" "$settings.bak"
        # Merge via jq — preserves existing keys
        jq -s '.[0] * .[1]' "$settings" "$snippet" > "$settings.tmp"
        mv "$settings.tmp" "$settings"
    else
        cp "$snippet" "$settings"
    fi
    echo "✓ Claude settings merged."
}

ensure_symlink() {
    local link="$1" target="$2"
    mkdir -p "$(dirname "$link")"
    [[ -e "$link" || -L "$link" ]] && rm -rf "$link"
    ln -s "$target" "$link"
}

install_copilot_cli() {
    ensure_symlink "$HOME/.copilot/skills" "$REPO_ROOT/plugins/pitt-skills/skills"
    echo "✓ Copilot CLI: ~/.copilot/skills → repo"
}

install_copilot_chat() {
    ensure_symlink "$HOME/.copilot/instructions" "$REPO_ROOT/.github/instructions"
    [[ -d "$REPO_ROOT/.github/prompts" ]] && \
        ensure_symlink "$HOME/.copilot/prompts" "$REPO_ROOT/.github/prompts"
    echo "✓ Copilot Chat: ~/.copilot/{instructions,prompts} → repo"
}

IFS=',' read -ra TOOL_LIST <<< "$TOOLS"
for tool in "${TOOL_LIST[@]}"; do
    case "$tool" in
        claude) command -v claude >/dev/null && merge_claude_settings || echo "claude not installed, skipping" ;;
        copilotCli) install_copilot_cli ;;
        vscode) install_copilot_chat ;;
    esac
done
echo "✓ Done."
```

### Task 31: Vendoring helper — `scripts/vendor-skill.ps1`

**Files:**

- Create: `tests/vendor/Vendor.Tests.ps1`

**Step 1: Write Pester tests**

```powershell
Describe "Vendor-Skill" {
    It "copies SKILL.md and creates UPSTREAM.md with metadata" {
        # Set up fake source dir...
        # Call Vendor-Skill -Source <path> -SkillName foo -UpstreamRepo "x/y" -UpstreamSha "abc"
```

**Step 2: Implement**

```powershell
[CmdletBinding()]
param(
    [Parameter(Mandatory)] [string] $Source,         # path to existing SKILL.md or its containing dir
    [Parameter(Mandatory)] [string] $SkillName,
    [Parameter(Mandatory)] [string] $UpstreamRepo,   # e.g., obra/superpowers
    [Parameter(Mandatory)] [string] $UpstreamSha,
    [string] $License = 'MIT',
    [string] $RepoRoot = (Split-Path $PSScriptRoot -Parent)
)

# Resolve $Source to a SKILL.md path
if (Test-Path (Join-Path $Source 'SKILL.md')) { $Source = Join-Path $Source 'SKILL.md' }
if (-not (Test-Path $Source)) { throw "SKILL.md not found at $Source" }

$dest = Join-Path $RepoRoot "plugins/pitt-skills/skills/$SkillName"
New-Item -ItemType Directory -Path $dest -Force | Out-Null
Copy-Item $Source (Join-Path $dest 'SKILL.md')

$upstream = @"
# Upstream source

- **Repo:** https://github.com/$UpstreamRepo
- **Commit SHA at vendoring:** $UpstreamSha
- **Original license:** $License
- **Vendored on:** $(Get-Date -Format 'yyyy-MM-dd')

## My changes

(none yet — record changes here as you make them)
"@
Set-Content (Join-Path $dest 'UPSTREAM.md') $upstream

Write-Host "✓ Vendored $SkillName from $UpstreamRepo@$UpstreamSha"
```

## Notes for the executing engineer

- **Skip-able skills you may want:** `superpowers:test-driven-development` (TDD discipline), `superpowers:writing-skills` (when authoring new skills in M2/M4), `superpowers:verification-before-completion` (after each milestone).
- **PowerShell versioning:** assume `pwsh` (PowerShell 7+) is available. `Windows PowerShell 5.1` will not work due to `ConvertFrom-Json -AsHashtable`.
- **Pester:** install with `Install-Module Pester -Scope CurrentUser -Force` if not present.
- **bats-core:** `npm install -g bats` or system package.
- **`gh` CLI:** required for Tasks 7 and 38; users without it can substitute web UI.
- **YAML parsing in `Parse-Frontmatter`:** intentionally simple — handles only the schema we author. If you author a skill with deep nesting or arrays, extend the parser at that point. Don't pre-build a full YAML parser.
- **Symlinks on Windows:** Developer Mode preferred; junction fallback works for directories without admin. The plan uses `mklink /J` as fallback. Files (not dirs) won't have a fallback — but the plan only ever symlinks directories.
- **Branch policy:** never commit to `main` directly past M1's initial scaffolding. Use feature branches per task or per milestone, PR into `main`. (User memory rule.)
- **Render API:** unrelated to this project; ignore.
